# 用戶體驗報告 (手機版模擬 - Round 4)

## 測試環境
- **設備**: Mobile Simulation (375x812)
- **測試員**: 非技術背景用戶 (模擬)

## 🛑 發現的各種問題 (Bug List)

### 1. Header 遮擋問題 (High Priority)
- **狀況**: 雖然已嘗試修復，但在捲動時，搜尋欄 (`Sticky Header`) 仍然給人「遮擋內容」的感覺，特別是當內容標題 (如 `## 6歲`) 被蓋住時。
- **用戶回饋**: "搜尋欄就把照片上面的文字遮掉一半了...感覺畫面有點擁擠。"
- **建議**: 可能需要再增加頂部間距，或檢查 CSS 變數是否正確生效。

### 2. 側邊欄無法點擊背景關閉 (Medium Priority)
- **狀況**: 打開側邊欄後，點擊右側的灰色遮罩區域沒有任何反應。
- **原因**: 用戶移除了 Sidebar Overlay 的相關 CSS 與 JS 邏輯。
- **用戶回饋**: "我習慣點一下右邊黑黑的部分來關閉它，結果沒反應！"

### 3. 設定頁面日期顯示錯誤 (Critical Data Bug)
- **狀況**: 涵涵的生日顯示為 `202101/01/05`，明顯格式錯誤。
- **用戶回饋**: "這寫錯了吧？"

### 4. 設定頁面輸入體驗不佳 (Usability)
- **狀況**: 日期欄位是純文字輸入，手機上難以操作。
- **建議**: 改用 `<input type="date">`。

---

## ✅ 體驗良好的部分
- **Logo 導航**:點擊左上角「黃家小屋」能順利回到首頁，符合預期。
- **搜尋欄隨動**: 用戶覺得隨時能搜尋很方便 (只要不遮擋內容)。

## 📝 綜合建議
建議優先修復 **設定頁面日期格式** (資料錯誤) 與 **側邊欄互動** (操作受挫)。Header 遮擋問題可視情況再微調間距。
